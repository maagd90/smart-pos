# Milestone 1 Review — Platform Scaffold

## Delivered in this branch

This milestone intentionally stops at scaffolding so the architecture can be reviewed before feature implementation.

### Added

- `microservices-platform/README.md` — review guide and local run instructions.
- `microservices-platform/service-catalog.yml` — canonical service names, ports, databases, and permissions.
- `microservices-platform/.env.example` — local environment values.
- `microservices-platform/docker-compose.yml` — Kafka, Redis, Eureka, API Gateway, one PostgreSQL database per service, and all domain services.
- `microservices-platform/scaffold-services.sh` — generates all runnable Spring Boot service skeletons, shared contracts, and web/mobile client skeletons.

## Architecture decisions captured

| Area | Decision |
|---|---|
| Architecture | Complete microservices, isolated per bounded context |
| Repository approach | Staged in this existing repo under `microservices-platform/`; service directories are designed to be split into polyrepos later |
| Gateway | Spring Cloud Gateway |
| Discovery | Eureka for local development; Kubernetes DNS later |
| Broker | Kafka |
| Database | PostgreSQL, one database per service |
| Migration | Flyway per service |
| Cache/rate limit foundation | Redis |
| Web client | React + TypeScript skeleton |
| Mobile client | React Native + Expo skeleton |
| External integrations | Stubbed in later milestones behind interfaces |
